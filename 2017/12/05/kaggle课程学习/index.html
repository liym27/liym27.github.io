kaggle课程笔记之week1

第一节

讲了kaggle概述

第二节

演示了网站上的kaggle的信息

第三节 Real Word Application vs Competitions

1.在机器学习之前，从商业角度理解问题
2.定义问题，收集数据
3.清洗数据，预处理
4.哪类模型适合这个问题，如何计算performance，如何选择最佳模型
5.部署模型，后期追踪修改模型等
6.不仅仅是关于算法，算法只是一个tool
7.不要限制自己，creativity

但是一般在比赛中，很多步骤已经做好了，我们需要做的是3,4

第四节 Recap of main ML algorithms

1.Linear model

比如：Logistic Regression, SVM
优点：即使在高维数据中，它们也表现的很好
缺点：非线性数据集上表现差
都是线性模型，loss function不一样这些模型在很多库中都有实现，最出名的Scikit-learn，指得我们注意的Vowpal Wabbit（它专门处理large data sets）

2.Tree-based

比如：Decision Tree, Random Forest, GBDT
优点：a good default method for tabular data(表格数据)
缺点：有时候边界无法正确分类
用库XGBoost, LightGBM,它们更快速更精确

3.KNN

找到一个中心，离中心近的分为一类
Sklearn

4.Neural Network

比起决策树更smooth
更多专门的框架需要了解。首推Pytorch

5.No free lunch，没有一种方法是万能的

因为每一种方法都对data或task做了假设，需要去理解各种算法是如何划分决策边界的

6.总结

• There is no “silver bullet” algorithm
• Linear models split space into 2 subspaces
• Tree-based methods splits space into boxes
• k-NN methods heavy rely on how to measure points“closeness”
• Feed-forward NNs produce smooth non-linear decision boundary.
The most powerful methods are Gradient Boosted Decision Trees and Neural Networks. But you shouldn’t underestimate the others.
