笔记1 之 创建用户、设置权限

全文可以查看 Linux 服务器上建立用户并分配权限

1.创建用户,然后根据提示输入密码 

sudo adduser newname

2.设置文件权限

sudo chmod 700 ××× （只有所有者有读和写以及执行的权限） *ym用这个*
sudo chmod 600 ××× （只有所有者有读和写的权限）
sudo chmod 644 ××× （所有者有读和写的权限，组用户只有读的权限）
sudo chmod 666 ××× （每个人都有读和写的权限）
sudo chmod 777 ××× （每个人都有读和写以及执行的权限）

3.传输文件

scp -P 端口号 源文件 目的地址  //外部传输需要 -P 端口号
scp 源文件 目的地址  //内部传输不需要端口号

4.安装anaconda

（1）安装

bash ***.sh  // 运行.sh包
// 然后一路 Enter下去，中间会让选择路径，一般我都用默认路径
//会选择是否默认添加PATH 到 .bashrc文件，一般默认yes

（2）测试是否安装成功

conda --version //注意重启后查看，不然的话

5.远程访问jupyter notebook

查看远程访问jupyter notebook

// 1.登录远程服务器
// 2.生成配置文件
    jupyter notebook --generate-config
// 3.1 生成密码
    In [1]: from notebook.auth import passwd
    In [2]: passwd()
    Enter password: 
    Verify password: 
    Out[2]: 'sha1:ce23d945972f:34769685a7ccd3d08c84a18c63968a41f1140274'
// 3.2 把生成的密文‘sha:ce…’复制下来
// 4. 修改默认配置文件
    $vim ~/.jupyter/jupyter_notebook_config.py
    // 进行如下修改，其实就是添加这几句话，因为配置文件全被注释了
    c.NotebookApp.ip='*'
    c.NotebookApp.password = u'sha:ce...刚才复制的那个密文'
    c.NotebookApp.open_browser = False
    c.NotebookApp.port =8888 #随便指定一个端口
// 5. 启动jupyter
    jupyter notebook
// 6. 远程访问
// 此时应该可以直接从本地浏览器直接访问http://address_of_remote:8888就可以看到jupyter的登陆界面。
// 7. 建立ssh通道
    如果登陆失败，则有可能是服务器防火墙设置的问题，此时最简单的方法是在本地建立一个ssh通道： 
    在本地终端中输入ssh username@address_of_remote -L127.0.0.1:1234:127.0.0.1:8888 
    便可以在localhost:1234直接访问远程的jupyter了。

笔记2 之 linux下监视NVIDIA的GPU使用情况

####1.显示当前GPU使用情况
Nvidia自带了一个nvidia-smi的命令行工具，会显示显存使用情况：

nvidia-smi

####2.周期性输出GPU使用情况
用 watch 命令，周期性执行 nvidia-smi
watch 基本用法： 

watch [options] command  # 最常见的参数是 -n,后面指定是每多少秒来执行一次命令
watch -n 10 nvidia-smi
