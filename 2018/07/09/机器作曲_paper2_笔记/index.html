Motivation

RL的优势：使得网络能够学习不可微的 reward function

模型 RL Tuner

整个模型是 RL Tuner。

  • NoteRNN: 训练data, 为 Q Network 和 Target Q Network 提供初始化权重。

  • reword定义：we define a set of music theory rules, and produce rewards based on whether the model’s compositions adhere to those rules. 其实就是根据音乐理论制定规则，如果生成的音乐符合规则的话，（我猜测）如果生成的音乐越符合规则，reword越高。

  • RewardRNN:用于计算概率值 —— NoteRNN预测了下一个note，RewardRnn计算它的播放概率。(我的理解)RewardRNN保留了NoteRNN的预测概率值，同时也加入了音乐理论的影响。

定义为一个RL问题：

  • action:选择下一个note;
  • state: (1)composition so far; (2) Q-network 与 Reward RNN 的内部LSTM状态；

  • reward fuction: 结合了 从data中学到的概率 和 音乐理论规则;

  • rMT(a,s) is calculated by a set of functions

  • Reward RNN is used to compute p(a|s), the probability of playing the next note a given the composition s as originally learned from actual songs.

Music Theory Rewards

基于已知的音乐理论（暂不扩展）

  • Stay in key：音符属于相同的key
  • Begin and end with the tonic note：作曲的第一个note 和 final bar 的第一个note应该是主音音符。
  • Avoid excessively repeated notes：避免过度重复。除非引入一个休息或一个音符被保持，否则一个音调不应该重复 4 次 in a row.
  • Prefer harmonious intervals：避免尴尬的时间间隔，如七分之一，或者一个大于八度的大跳跃。
  • Resolve large leaps：两次向相同的方向跳跃的话，reward是负面的。
  • Avoid continuously repeating extrema notes：避免不断重复极值音符。
  • Avoid high auto-correlation：避免高自相关。
